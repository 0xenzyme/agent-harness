# Agent Harness v0.10.0

Released: 2026-07-21

## Summary

Agent Harness now derives completion from authoritative Task/Goal state,
verification, required durable gates, and state synchronization. Runs retain
execution and verification evidence; the configured status file remains a
bounded projection.

## Completion Contract

- Task/Goal phase is `active`, `completed`, or `blocked`.
- `blocked` is resumable and non-complete.
- Run/DAG state records execution evidence rather than a second Task/Goal
  completion authority.
- State Sync Notes remain part of durable completion.
- New Goal and Run artifacts contain no Delivery State or Run-start checkout
  snapshot fields.

## Compatibility

Goals and Runs created before `0.10.0` may still contain `Delivery State`,
`deliveryState`, `deliveryPolicy`, `startHead`, `startBranch`, `startUpstream`,
or `startDirtyState`. The `0.10.0` CLI can read those artifacts, ignores the
legacy fields in validation and status output, and removes them when recording
an updated legacy Run. Historical artifacts are not rewritten automatically.

## CLI And Artifact Behavior

- `doctor`, work-mode recommendation, maintenance, Run prepare/record/status,
  and completion gates use Harness artifacts and configured policy.
- Artifact inspection scans configured durable-evidence roots for local Run
  references.
- New canonical templates and generated prompts describe accepted state,
  verification, gates, and state sync directly.

## Verification

The release gate requires:

```bash
npm run test:all
npm run test:eval
npm run validate:plugin
```

These checks cover canonical generation, legacy read compatibility, Run/DAG
completion, containment, bounded artifact lifecycle, documentation alignment,
and plugin structure.
