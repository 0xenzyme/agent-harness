# Agent Harness v0.6.0

Agent Harness `0.6.0` tightens state discipline after the `0.5.0`
cybernetic-stability line.

## Highlights

- Made task completion include state-sync evidence or `State Sync Notes`.
- Preserved the accepted-state owner boundary: workers and executing lanes
  return candidate state notes; the accepted-state owner verifies and records
  accepted state.
- Added `harness-rule:bounded-status-snapshot` so configured status files are
  bounded current-state snapshots, not append-only history logs.
- Updated README / README.zh-CN, capability matrix, project contract,
  workflow guidance, templates, generated prompts, CLI docs, and deterministic
  checks to carry the new state-sync and bounded-status semantics.
- Kept historical completion evidence in `harness/tasks.md`, `harness/goals/`,
  `.harness/runs/`, and gate records.

npm run test:presentation

## Delivery Notes

This version line does not add plugin hooks, daemons, watchers, telemetry, paid
services, production access, deployment automation, or release automation.
It aligns the completion contract and status-file contract so future Harness
execution can finish with explicit state-sync notes while keeping the live
status surface compact and current.
