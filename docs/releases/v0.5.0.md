# Agent Harness v0.5.0

Agent Harness `0.5.0` introduces the cybernetic stability design line.

## Highlights

- Added the dedicated [Cybernetic Stability Model](../cybernetic-stability.md).
- Framed Harness as a semantic control loop:
  `intent -> setpoint -> sensor -> measurement -> gap -> controller -> action -> feedback`.
- Documented intent recognition as target / setpoint selection.
- Documented sensor freshness, measurement snapshots, remaining gap,
  feedback quality, stability, and saturation as the next stability layer.
- Promoted the model into stable protocol anchors across task routing,
  workflow skills, goal/spec/worker templates, generated goal/run/prompt
  guidance, and deterministic checks.
- Updated README positioning so the version line explains the control-theory
  inspired upgrade.

## Verification

```bash
npm run test:presentation
npm run test:protocol
npm run test:smoke
npm run validate:plugin
git diff --check
```

## Delivery Notes

This version line does not add plugin hooks, daemons, watchers, telemetry, paid
services, production access, deployment automation, or release automation.
It strengthens the Harness control model, documentation surface, workflow
guidance, templates, generated handoff text, and deterministic checks so future
execution can reduce stale-state mistakes, false completion, route oscillation,
and weak feedback loops.
