# Agent Harness v0.9.0

Agent Harness `0.9.0` bounds the runtime artifacts produced by long-running
controller work while preserving the evidence needed for recovery and audit.

## Artifact Policy

- `artifactPolicy.runs` declares whether Run directories are tracked or kept
  local-only.
- Retention windows distinguish completed and blocked Runs and protect the
  configured number of latest Runs.
- Configured durable-evidence roots define where prune may resolve Goal and
  state-sync evidence.
- Status snapshots and task indexes have explicit bounds, while displaced task
  blocks can be retained exactly in an archive.

Projects without `artifactPolicy` keep compatible defaults. Fixed and adapter
templates expose the same policy surface without introducing project-specific
assumptions.

## Deterministic Lifecycle Commands

```bash
agent-harness artifacts inspect --cwd .
agent-harness artifacts compact --cwd .
agent-harness artifacts compact --cwd . --record
agent-harness artifacts prune --cwd .
agent-harness artifacts prune --cwd . --apply
```

- `inspect` is read-only and reports bounds, task state, Run count/bytes/phases,
  tracked references, and lifecycle inconsistencies.
- `compact` previews by default. `--record` archives exact task blocks before
  replacing the active index.
- `prune` previews by default. `--apply` only removes local-only terminal Runs
  that are old enough, outside latest-Run protection, contained by configured
  roots, referenced by a configured Goal, and backed by non-empty State Sync
  Notes.
- Active, unknown, uncontained, or insufficiently evidenced Runs are retained.

## Repository Dogfood

The repository now uses a bounded task index and status snapshot. Exact
historical task records moved to `harness/tasks-archive.md`; all existing Run
directories were retained. Existing tracked references to local-only Runs stay
visible as an audit warning instead of being silently discarded.
