# Capture Boundary

Use this reference when deciding whether a user message belongs in intake.

## Owns

- rough ideas
- requirements
- bug reports
- Idea Inbox notes
- capture-thread promotions
- triage into title, priority, section, related artifacts, and acceptance hint

## Does Not Own

- read-only project orientation
- harness setup or adoption
- confirmed implementation
- goal/run preparation
- branch, worktree, PR, deployment, or release work

## Preserve Raw Intent

Keep the raw note available. Summarize only what is needed for the task index.
Do not turn rough language into accepted scope without user or control-lane
confirmation.

## Record Boundary

Preview first. Record only when the user explicitly asks to add, save, record,
or promote the item.
