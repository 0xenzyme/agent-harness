# Capture And Promotion

Preserve the raw request, identify duplicates and dependencies, and preview a
title, priority, section, acceptance hint, and likely Spec need. Recording the
candidate requires explicit authority and does not accept scope.

Promotion requires clear objective, constraints, non-goals, verification, and
pause conditions. Ordinary accepted change/build returns to Codex directly;
use durable Harness execution only when recovery, audit, persistent state sync,
milestone/DAG, multi-worker, or high-risk control is required.
